## 🧱 Core, Actively Maintained Data Providers (Paid + Free Tiers)

### **FantasyData — Historical NCAA Odds (CSV/XLS)**
Consensus game‑level odds with downloadable CSV/XLS files. Access requires an All‑Access plan.
People access it directly via CSV downloads; no public scraper needed.
  [FantasyData](https://fantasydata.com/ncaa-basketball/odds)

### **The Odds API — JSON API for Current & Historical Odds**
Supports multiple bookmakers (US, EU, AU). Provides moneyline, spreads, totals.
Community uses Python/R wrappers and cron jobs to store snapshots.
  [The Odds API](https://the-odds-api.com/sports-odds-data/ncaa-basketball-odds.html)

### **TeamRankings — NCAA Odds & Line History**
Provides closing line history and spread distributions.
Access is via web scraping (community scrapers exist) or premium subscription.
  [TeamRankings.com](https://www.teamrankings.com/ncb/odds-history/results/)

### **Scottfree Analytics — NCAAB Historical Odds CSV (Paid)**
132‑column machine‑learning‑ready dataset with spreads, moneylines, totals, standardized team names, and final odds recorded within one hour of game time.
People buy once and integrate into pipelines; no scraping required.
  [scottfreellc.com](https://www.scottfreellc.com/shop/p/ncaab-historical-odds-data)

## 🧭 How People Actually Build 2018–2025 Historical Odds Datasets

Most practitioners combine:

1. **Pinnacle openers** (scraped or purchased)
2. **Consensus closers** (FantasyData, TeamRankings, Odds API)
3. **Book‑specific lines** (DraftKings, FanDuel, BetOnline)
4. **Manual backfilling** from community CSV dumps
5. **Metadata** from Sports‑Reference / Kaggle
6. **Line‑movement logs** from GitHub scrapers or Discord archives

This produces a unified dataset with:
- Opening ML/spread/total
- Closing ML/spread/total
- Book‑by‑book availability
- Standardized team names
- Game metadata (date, location, neutral site, etc.)

### 2. Commercial & semi-commercial odds providers

These are what serious modelers usually anchor on, then backfill with scrapes.

#### The Odds API

- **What it is:** JSON API with current and some historical NCAA basketball odds from multiple US-facing books.   [The Odds API](https://the-odds-api.com/sports-odds-data/ncaa-basketball-odds.html)
- **Opening vs closing:**
  - Strong for *current* and recent lines; historical depth for 2018–25 is more limited and often closer to “closing/last available” than true openers.
  - Some users snapshot odds daily to build their own historical archive.
- **How people access it:**
  - **Official:** API key → REST calls → Python/R/Sheets.
  - **Community:** Small Python wrappers, Airflow/cron jobs writing to Postgres/BigQuery.
- **Scrapers:**
  - Not needed for the official API, but people do build **rate-limit-aware collectors** to store every refresh and reconstruct line movement.

#### SportsDataIO (NCAA College Basketball + Vault)

- **What it is:** Full-stack sports data provider with NCAAB scores, stats, and betting odds; “Vault” product for historical data.   [SportsDataIO](https://sportsdata.io/developers/api-documentation/ncaa-basketball)
- **Opening vs closing:**
  - Offers historical odds and betting lines; higher tiers include **line movement** (open, intermediate, close) across books.
- **How people access it:**
  - **Official:** Paid API (JSON) + bulk historical exports via Vault.
  - **Community:** Used heavily by professional/serious hobby modelers; often mentioned in analytics Discords as “pay but worth it if you’re serious.”
- **Scrapers:**
  - No need to scrape; instead, people write **ETL pipelines** around the official API/exports.

#### FantasyData NCAA Basketball Odds

- **What it is:** Consensus game-level odds with CSV/XLS downloads; historical odds available with All-Access plan.   [FantasyData](https://fantasydata.com/ncaa-basketball/odds)
- **Opening vs closing:**
  - Typically provides **pre-game lines**; depending on plan, you can get open/close and sometimes movement.
- **How people access it:**
  - Web interface with filters → CSV/XLS export; some use scheduled downloads.
- **Scrapers:**
  - Light HTML scrapers exist (Python/BeautifulSoup) to automate pulling daily odds pages when CSV export is too manual.

#### BigDataBall NCAAB datasets

- **What it is:** Season-by-season NCAAB spreadsheets with stats and odds.   [BigDataBall](https://www.bigdataball.com/datasets/ncaa/cbb-data/)
- **Opening vs closing:**
  - Typically includes **pre-game lines** (often closing or consensus); some seasons have book-level lines.
- **How people access it:**
  - One-time purchases per season (e.g., 2022–23, 2023–24, 2024–25) as CSV/XLS.
- **Scrapers:**
  - None needed; people just download and version-control the files.

### 3. Community scrapes, GitHub repos, and “obscure but real” sources

These are where you get closer to what the community *actually* hacks together.

#### OddsPortal / Covers / VegasInsider / DonBest-style sites

- **What they are:** Long-running odds comparison sites with historical lines, often including open/close and line movement by book.
- **Opening vs closing:**
  - Frequently have **openers, closers, and intermediate moves**; coverage back to at least 2018 for major conferences and tournaments.
- **How people access them:**
  - **Unofficial:** HTML scraping (Python, R, or Node) with rotating user agents and backoff.
  - Some users manually export tables or copy-paste into spreadsheets for specific seasons.
- **Scrapers:**
  - Multiple GitHub repos exist that:
    - Crawl game pages by date/season.
    - Parse odds tables into structured data (game_id, book, open, close, timestamps).
  - Many are semi-broken and require fixing CSS selectors or URL patterns each season.

#### Action Network, BettingPros, Rotowire, etc.

- **What they are:** Media/analytics sites that show live and historical odds, sometimes with “line history” charts.
- **Opening vs closing:**
  - Often show **open and current** lines; some show full movement.
- **How people access them:**
  - Mostly via web UI; some power users scrape the underlying JSON endpoints used by the front-end.
- **Scrapers:**
  - GitHub projects and private scripts that hit undocumented JSON endpoints (e.g., `/api/odds?gameId=...`) and store snapshots.

#### GitHub search patterns that actually surface useful stuff

If you search GitHub for:

- **“NCAAB odds scraper”**, **“college basketball odds scraper”**, **“oddsportal ncaab”**, **“vegasinsider ncaab scraper”**, **“sports betting scraper”**, you’ll find:
  - **One-off season scrapes** (e.g., “NCAAB 2019 odds from OddsPortal”).
  - **Generalized scrapers** that support multiple sports but include `basketball_ncaab` as a sport key.
  - **R packages** that wrap betting APIs or scrape odds sites.

Most of these are:

- Partially deprecated (selectors broken).
- Still extremely valuable as **templates** for your own scraper.

### 4. Academic, Kaggle, and “side-door” datasets

#### Kaggle college basketball datasets

- **What they are:** Large NCAAB datasets (2013–2025) focused on team stats, efficiency, and results; some forks add odds.   [Kaggle](https://www.kaggle.com/datasets/andrewsundberg/college-basketball-dataset)
- **Opening vs closing:**
  - Base dataset is mostly stats; some community versions add **closing spreads/totals** from scraped sources.
- **How people access it:**
  - Direct CSV download; many use Kaggle notebooks to join with their own odds data.
- **Scrapers:**
  - Not needed; but you’ll see notebooks that merge Kaggle stats with scraped odds from OddsPortal/Covers.

#### Academic / research-oriented sources

- **NCAA tournament prediction papers** sometimes publish:
  - CSVs with **point spreads and totals** for tournament games (often closing lines only).
  - These are usually limited to March Madness, not full regular seasons.
- **How people access them:**
  - Links in papers, personal websites, or OSF/Zenodo repositories.
- **Scrapers:**
  - None; you just download the CSVs and treat them as partial coverage.

### 6. Practical strategy for your 2018–2025 NCAAB model

Given what you’re trying to build, here’s a workflow that matches how serious modelers usually do it:

1. **Decide your “truth” source for closing lines.**
   - **Option A:** SportsDataIO or FantasyData as your canonical closing line provider (paid, clean).
   - **Option B:** OddsPortal/Covers scrape as your canonical source if you want to avoid subscription costs.

2. **Layer in opening lines + line movement.**
   - Use:
     - SportsDataIO line-movement endpoints (if you pay for that tier).   [SportsDataIO](https://sportsdata.io/developers/api-documentation/ncaa-basketball)
     - Or your own **high-frequency scraper** of a site that shows open + movement (OddsPortal, Action Network, etc.).

3. **Backfill missing games and edge cases.**
   - Use BigDataBall and FantasyData CSVs to patch gaps, especially for smaller conferences or early-season games.   [FantasyData](https://fantasydata.com/ncaa-basketball/odds)  [BigDataBall](https://www.bigdataball.com/datasets/ncaa/cbb-data/)

4. **Use Kaggle and academic datasets for non-odds features.**
   - Team strength, tempo, efficiency, etc., from Kaggle and BartTorvik/KenPom-style sources; join on date/team to your odds.   [Kaggle](https://www.kaggle.com/datasets/andrewsundberg/college-basketball-dataset)

5. **Continuously mine GitHub + Reddit/Discord for updated scrapers.**
   - Treat public scrapers as **starting points**, not production code.
   - Fork them, fix selectors, add logging, and build a reproducible ETL around them.
